# Guardrail Checklist (BlenderMCP)

このチェックリストは、Commander / ClaudeCode / Codex の
すべてのエージェントが各サイクルの前後に必ず確認すべき共通基準である。

---

# 0. New Project（新規プロジェクト開始時）

- [ ] ユーザーの入力から要件を抽出し、整理して確認を返したか
- [ ] プロジェクト名（snake_case）、用途、概要、サイズ、スタイル、制約を明示したか
- [ ] 不明点があれば質問したか（最大3件）
- [ ] Commander承認後に `scripts/new_project.py` で子プロジェクトを生成したか
- [ ] Blender接続確認（`get_scene_info`）を行ったか
- [ ] シーン初期化を実行したか
- [ ] 初期 `.blend` を `projects/{name}/models/wip/` に保存したか

---

# 1. Context（文脈の正しさ）

- [ ] Context Block が冒頭に存在しているか
- [ ] BasedOn の対象（前回出力 / ファイル）が正しいか確認
- [ ] Summary が"コピペではなく"自分の言葉で再構築されているか
- [ ] Objective が明確で1行になっているか
- [ ] 前のサイクルと矛盾がないか

---

# 2. Relay（文脈引き継ぎ）

- [ ] Relay Header が末尾に存在するか
- [ ] NextTarget が正しいエージェントになっているか
- [ ] ExpectedFocus が「何を見るべきか」をはっきり示しているか
- [ ] 次のAIがこの内容で迷わず動けるか（曖昧さがないか）

---

# 3. Scene Management（シーン管理）

- [ ] 操作前に `get_scene_info` で現状を確認したか
- [ ] 操作後に `get_viewport_screenshot` で結果を確認したか
- [ ] `execute_blender_code` 使用前に `.blend` を保存したか
- [ ] 1指示 = 1操作の原則を守っているか
- [ ] オブジェクト命名規則（英語 snake_case）に従っているか
- [ ] マテリアル命名規則（用途_色）に従っているか

---

# 4. Implementation（ClaudeCode 向け）

- [ ] Task をサブタスク化し、実行可能な形に落とし込んだか
- [ ] 各操作で使用したMCPツールが明示されているか
- [ ] 結果（スクリーンショット・変更内容）が提示されているか
- [ ] 不要な変更・独断の仕様変更をしていないか
- [ ] 再利用可能な操作は `scripts/` に保存したか
- [ ] 作業の節目で `models/wip/` にバージョン番号付き保存したか

---

# 5. Review（Codex 向け）

- [ ] MUST / SHOULD / CONSIDER の指摘が分類されているか
- [ ] 指摘内容が「事実ベース」で明確か
- [ ] 用途別チェック（3Dプリント/ゲーム/建築）を実施したか
- [ ] Verify（どう確認すべきか）が書かれているか
- [ ] Commander に判断を委ねるべき点が明確化されているか

---

# 6. Quality（3Dモデル品質）

## 3Dプリント用
- [ ] 最小肉厚 1.5mm（FDM）を確認したか
- [ ] 非マニフォールドエッジがゼロか
- [ ] オーバーハング 45度以下を確認したか
- [ ] STL エクスポートが正常か

## ゲームアセット用
- [ ] ポリゴン数が目標以下か（キャラ~5000、小物~500）
- [ ] UV展開に重なりがないか
- [ ] 原点がオブジェクト底面中心にあるか
- [ ] GLB エクスポートが正常か

## 建築・工業モデル用
- [ ] スケールが正確か（1単位=1メートル）
- [ ] 寸法精度（小数点2桁）を満たしているか

---

# 7. Files & Naming（ファイル管理）

- [ ] 保存先が正しいディレクトリか（projects/{name}/models/wip, renders/, exports/）
- [ ] バージョン番号（_v001, _v002）が付いているか
- [ ] エクスポート形式が用途に合っているか（STL/GLB/FBX）
- [ ] 不要な一時ファイルが残っていないか

---

# 8. Git（コミット・ブランチ運用）

- [ ] コミットメッセージが分かりやすいか（Conventional Commits 推奨）
- [ ] ブランチ命名規則に従っているか
- [ ] main へ直接 push していないか
- [ ] .blend ファイルが Git LFS で管理されているか（該当する場合）

---

# 9. Risk（リスク管理）

- [ ] メッシュ品質・スケール・座標系の整合性が確認されたか
- [ ] エクスポート互換性のリスクが明示されたか
- [ ] 質問は最大3件に絞られているか

---

# 10. Completion（サイクル完了条件）

- [ ] 全AIの出力が Context Block と Relay Header を持っている
- [ ] Commander が「承認 / 保留 / 再実行」を判断した
- [ ] 必要なファイル保存・エクスポートが済んだ
- [ ] 次サイクルのタスクが明確化された
